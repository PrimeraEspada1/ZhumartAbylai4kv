# Django Lab Setup

Этот проект содержит отдельную Django-часть для лабораторных работ 4 и 5.

## 1. Установить зависимости

```powershell
pip install -r requirements.txt
```

или

```powershell
py -m pip install -r requirements.txt
```

## 2. Выполнить миграции

```powershell
py manage.py makemigrations
py manage.py migrate
```

Если `0001_initial.py` уже используется без изменений, достаточно:

```powershell
py manage.py migrate
```

## 3. Создать суперпользователя

```powershell
py manage.py createsuperuser
```

## 4. Запустить сервер

```powershell
py manage.py runserver
```

## 5. Открыть проект в браузере

- Домашняя: `http://127.0.0.1:8000/`
- Регистрация: `http://127.0.0.1:8000/register/`
- Вход: `http://127.0.0.1:8000/login/`
- Профиль: `http://127.0.0.1:8000/profile/`
- Заявки: `http://127.0.0.1:8000/requests/`
- Админка: `http://127.0.0.1:8000/admin/`

## 6. При желании загрузить готовые заявки для демонстрации

```powershell
py manage.py loaddata accounts/fixtures/service_requests.json
```

или через полный путь к Python, если `py` не работает.

## Что реализовано

- главная страница;
- страница регистрации;
- страница входа;
- модель профиля пользователя в БД;
- регистрация модели в админке;
- фото профиля только для текущего пользователя;
- загрузка только файлов изображений;
- хранение файлов по отдельным папкам пользователей;
- фото по умолчанию, если пользователь ничего не загрузил.
- общая навигация через базовый шаблон;
- модель заявок на ремонт по теме проекта;
- список заявок и страница подробной информации;
- светлая и тёмная тема для улучшения UX.
